# Cloudinary Storage Folder Strategy

This document defines the remote folder structures and upload configurations for Maddy BGMI Store V2. This strategy keeps media assets organized and optimized.

---

## 1. Directory Structure

All media uploads must be routed to specific folders under the primary workspace directory.

```text
maddy-bgmi-store/
├── accounts/          ← Preview images of BGMI listing inventories (Array of strings)
├── reviews/           ← Customer review screenshots uploaded via public reviews page
├── payment-proofs/    ← Transaction screenshots showing proof of payment uploaded by admins
├── banners/           ← Hero banners, services banners, and structural layout assets
└── users/             ← User avatar images uploaded during Clerk profile configuration
```

---

## 2. Upload Presets Configuration

| Folder | Upload Preset Name | Access Mode | Transformations & Optimizations |
| :--- | :--- | :--- | :--- |
| `accounts/` | `mbs_accounts` | Public | - Format: Auto-converted to `WebP` or `AVIF` for quick loading.<br>- Quality: `auto:good` compression.<br>- Resizing: Auto-scale to max width `1200px` for inventory grids. |
| `reviews/` | `mbs_reviews` | Public | - Format: Auto `WebP`. Max size limit `5MB` per upload.<br>- Resizing: Auto-scale to width `800px` to fit feedback displays. |
| `payment-proofs/`| `mbs_proofs` | Gated (Admin-only) | - Format: Native `PNG` or `JPG` to preserve details.<br>- Quality: `auto:best` for legal audit preservation. |
| `banners/` | `mbs_banners` | Gated (Admin-only) | - Format: Auto `AVIF`/`WebP`. Compression `auto:good`. |

---

## 3. Upload Implementation Rule

1. **Client Upload Signing**: Direct client-side uploads to Cloudinary must utilize a secure, server-signed request generated by a Next.js Server Action in `packages/lib/src/cloudinary.ts`. This verifies that:
   - The user is authenticated.
   - The upload target is restricted to the correct subdirectory.
   - The file size does not exceed the preset limits (e.g. 5MB for reviews, 10MB for accounts).
2. **Metadata Naming**: Files must be renamed dynamically during upload to include identifiers:
   - Accounts: `acc_[product_uuid]_[timestamp]`
   - Reviews: `rev_[review_uuid]_[timestamp]`
   - Proofs: `proof_[transaction_id]_[timestamp]`
